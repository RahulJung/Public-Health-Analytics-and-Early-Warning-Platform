# Public Health Surveillance Analytics Project Summary

## Project Overview

The Public Health Signal Detection and Response Dashboard is an independent public health informatics research prototype for syndromic surveillance analytics modernization. It demonstrates how public aggregate or synthetic emergency department surveillance data can be converted into analyst-facing operational intelligence through ETL, feature engineering, anomaly detection, forecasting, geospatial risk scoring, data quality monitoring, synthetic HL7 simulation, and retrieval-augmented public health search.

The current implementation uses CDC-derived emergency department visit trajectory data for three respiratory syndrome categories:

- COVID-like illness
- Influenza-like illness
- RSV-like illness

The application is implemented as a Streamlit dashboard with modular Python components under `src/`.

## Problem Statement

Public health agencies and analysts need timely, explainable, and scalable tools for identifying unusual patterns in emergency department visit data. Surveillance workflows often require analysts to monitor multiple jurisdictions, syndrome categories, model outputs, and data quality indicators at the same time.

This prototype addresses that challenge by demonstrating a full analytics workflow that converts raw public/synthetic surveillance data into:

- Baseline comparisons
- Anomaly alerts
- Forecast trends
- Geographic risk rankings
- Alert intelligence tables
- Data quality indicators
- Natural-language surveillance summaries

## Public Health Context

Syndromic surveillance supports early detection and situational awareness by monitoring symptoms, visit patterns, and related indicators before confirmed diagnoses are available. It is useful for respiratory illness monitoring, outbreak detection, overdose surveillance concepts, heat illness and environmental health monitoring, emergency preparedness, and cross-jurisdiction public health intelligence.

This project is conceptually aligned with public health modernization themes reflected in:

- CDC National Syndromic Surveillance Program
- CDC ESSENCE
- CDC BioSense Platform
- CDC Data Modernization
- HHS emergency preparedness priorities
- WHO epidemic intelligence concepts

The prototype is not affiliated with, endorsed by, or operated by any government agency.

## Methods

The system includes:

- Data ingestion from public CDC-derived files or local cached CSV data.
- Cleaning and standardization of dates, geography, syndrome categories, and visit percentage values.
- Wide-to-long transformation of syndrome percentage fields.
- State-level aggregation from county-level source records.
- Feature engineering with rolling baseline, standard deviation, z-score, lag values, percent change, and trend acceleration.
- Z-score anomaly detection.
- Isolation Forest anomaly detection.
- Short-term forecasting with ARIMA, Prophet when available, Exponential Smoothing fallback, and rolling-average fallback.
- Composite risk scoring based on severity, growth, anomaly status, and persistence.
- Geospatial visualization using state-level risk summaries.
- Rule-based chief complaint classification.
- RAG-based public health query assistant over local documentation and live surveillance context.
- Synthetic HL7 ADT-style message generation for testing and demonstration.

## Operational Readiness Considerations

Before operational use, a system like this would require:

- Secure authentication and role-based access control
- PHI governance and privacy review
- Audit logging and alert disposition tracking
- Model versioning and threshold governance
- Data lineage and reproducibility controls
- Drift monitoring and calibration
- Human review workflow
- HL7/FHIR interoperability design
- Cloud deployment, monitoring, and incident response planning

## Limitations

- Uses public aggregate or synthetic data rather than live clinical feeds.
- Visit counts are unavailable in the current public percentage-only source.
- Current syndrome scope is limited to COVID-like illness, influenza-like illness, and RSV-like illness.
- County-level raw records are averaged to state-level trends, which may hide local hotspots.
- Precision and recall require adjudicated ground truth labels and are not asserted from the current unlabeled dataset.
- Forecasting is a prototype analytics module and should not be interpreted as an operational prediction system.
- NLP classification is rule-based and requires validation before operational use.
- RAG responses depend on retrieved local context and should be reviewed by a human analyst.
- The system is not an official HHS, CDC, WHO, NSSP, BioSense, ESSENCE, clinical, or public health decision system.
